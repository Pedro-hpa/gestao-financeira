# Checklist para finalizar no portfólio

## Antes de publicar no GitHub

- Remover a pasta `env/` do repositório.
- Remover `db.sqlite3` do repositório.
- Manter `requirements.txt`, `.gitignore`, `.env.example` e `README.md`.
- Criar um repositório novo no GitHub com nome profissional, por exemplo: `gestao-financeira-django`.
- Subir apenas o código-fonte limpo.

## Descrição curta para o portfólio

Sistema de gestão financeira pessoal desenvolvido em Django, com autenticação de usuários, dashboard financeiro, cadastro de receitas/despesas, filtros por período e categoria, além de operações de edição e exclusão protegidas por usuário.

## Pontos fortes para destacar

- CRUD completo de transações financeiras.
- Autenticação e separação dos dados por usuário.
- Validações no formulário de cadastro de transações.
- Dashboard com indicadores financeiros consolidados.
- Código organizado com models, forms, views, templates e migrations.

## Melhorias recomendadas para uma próxima versão

- Gráfico de evolução mensal.
- Exportação de dados para CSV ou PDF.
- Categorias personalizadas por usuário.
- Deploy em Render, Railway ou PythonAnywhere.
- Testes automatizados para views e models.
